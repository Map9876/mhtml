## 知見

- **コーディング知見**: [docs/knowhow/plugin-infrastructure-patterns.md](../../docs/knowhow/plugin-infrastructure-patterns.md) に「DOM 後処理プラグイン」節を追加 — Summary に `url` 無し / cheerio.text() の二重デコード回避 / 制御文字一括エスケープ / 404 リトライ深さ制限 / API 失敗の静か fallback の 5 点
- **タスク総括**:
  - **`enrichWithXxx(summary, $, landingUrl)` パターン** の繰り返し適用で、cheerio フィクスチャテストを vi.mock なしで書ける。phase3.1 の `buildSummaryFromOEmbed` パターンと完全に対称
  - **「Summary に url が無い」気づきは Plan 段階で予測されていた** (リスク・注意点 #2 の `parseGeneral` 戻り値設計の議論) が、実装中に再認識して `tryFetch` の戻り値を `{summary, usedUrl}` にする形で対応
  - **Edit ツールが正規表現中の制御文字を表示で失う**問題: `[\x00-\x1F]` のような可読な書き方を最初から使うべき（生制御文字を貼ると Edit が壊れる）
